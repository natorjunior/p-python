Funções na Matemática

Entrada -> Regra -> Saída

f(x) = x

f(10) = 10

g(x) = 2x + 3

g(10) = 23

h(x) = x²

h(10) = 100

Em programação, funções seguem a mesma ideia: recebem entrada, processam e geram saída.

Pergunta Provocativa

Pense como alguém que vai manter esse código no futuro.

Por que repetir código é ruim?

Se a regra mudar, vamos precisar corrigir em vários lugares. Isso aumenta retrabalho e erros.

Repetição de Código

Mesmo cálculo escrito duas vezes.


nota1, nota2 = 8.0, 6.0
media1 = (nota1 + nota2) / 2
print("Média aluno 1:", media1)

nota3, nota4 = 7.5, 9.0
media2 = (nota3 + nota4) / 2
print("Média aluno 2:", media2)

A mesma lógica foi escrita duas vezes. Em turmas grandes isso cresce rapidamente.

Reutilização com Funções

Uma função, várias chamadas.


def calcular_media(a, b):
    return (a + b) / 2

print("Média aluno 1:", calcular_media(8.0, 6.0))
print("Média aluno 2:", calcular_media(7.5, 9.0))

Agora a regra está centralizada em uma função, fácil de manter e reutilizar.

Passo a Passo (Funções)

  1. Passo 1: def indica que estamos definindo uma função.
  2. Passo 2: funcao é o nome usado para chamar essa função.
  3. Passo 3: os parênteses () guardam os parâmetros.
  4. Passo 4: a linha identada é o bloco de instruções da função.

Atividade A1 - Estrutura de Função

Identificando as partes


def saudacao(nome):
    print(f"Olá, {nome}")

Reescreva a função com o nome boas_vindas.

  1. Passo 1 (programa): executa print("Antes da função").
  2. Passo 2 (programa): faz a chamada funcao().
  3. Passo 3 (função): executa print("Olá mundo").
  4. Passo 4 (função): termina e faz o retorno para o programa.
  5. Passo 5 (programa): continua em print("Depois da função").

Regra prática: o principal executa, chama, pausa; a função executa e retorna; então o principal continua.

Auto animate - quadro 1: global e local no mesmo código.
